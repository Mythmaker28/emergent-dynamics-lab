# DATA_SCHEMA — portable exports (release/data/)

All numbers derive from committed raw pickles (retained in-repo for provenance). These CSV/JSON/NPZ files
let a reader reproduce every primary figure/table without unpickling. SHA-256 of each raw input is in
`data_manifest.json`.

## holdout_longitudinal.csv  (long format; 36 records × 4 checkpoints = 144 rows)
Source: `results/observer/tca_holdout_raw.pkl` (untouched sealed seeds 38502–38504; 12 sealed histories × 3 seeds).
| column | meaning |
|---|---|
| seed | simulation seed (38502/38503/38504); disjoint from all development seeds |
| history | history index hi (0–11); the grouping/donor unit for leave-one-history-out CV |
| h1_true | true cumulative-dose coordinate (a_early + a_late) for this history |
| h2_true | true temporal-order coordinate (a_late − a_early) for this history |
| step | integration step at the checkpoint (0, 400, 650, 800) |
| M | surviving original-material fraction at the checkpoint (1.0 → ~0.19) |
| size | tracked-entity size (cells) |
| mminus_std | dispersion (std over entity cells) of m_minus = m1 − m2 |
| mean_m1 … p90_m2 | 10-D memory order-statistics [mean,std,p10,p50,p90] of m1 then m2 over entity cells |

## holdout_deep_arrays.npz  (deep checkpoint, step 800)
Keys: `features` (36×11: the 10-D stats + mminus_std), `h1_true` (36), `h2_true` (36), `history` (36),
`seed` (36), `feature_names` (11). This is the exact input to the primary decoder.

## causal_transplant.csv / causal_inplace.csv
Source: `results/wd01_phasec/phasec_causal_{transplant,inplace}_raw.pkl`. Per (history, seed) rows with the
true (h1,h2) and the L2 norms of the multi-axis response vectors under the full model and under the
memory-inert (`both0`) / erased (`erase`) ablations. These support the causal necessity check (inert/erase
carry no history information).

## primary_table.csv  (regenerated by python -m reproduction.primary)
Per-checkpoint held-out decode: `mean_M, h1_R2, h1_ci_lo, h1_ci_hi, h2_R2, h2_ci_lo, h2_ci_hi`.

## primary_results.json / data_manifest.json
Machine-readable results, gate outcomes, seeds, checkpoint list, and SHA-256 of every raw input.

## Coordinate & decoder conventions
- Decoder: grouped leave-one-history-out ridge, λ = 1.0, per-fold standardization (constant columns dropped),
  features = 11-D above; R² = pooled coefficient of determination over held-out records.
- Uncertainty: donor-level (history-grouped) percentile bootstrap, n_boot = 3000, fixed seed 20260715.
- Certification threshold: held-out R² lower bound > 0.50.
