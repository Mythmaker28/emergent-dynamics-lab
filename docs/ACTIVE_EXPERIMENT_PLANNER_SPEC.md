# ACTIVE EXPERIMENT PLANNER — SPECIFICATION (frozen; RETIRED at D-069)

> Do not ask every possible question. Ask the smallest number of legal questions that force the remaining causal
> explanations apart — and know when the world does not permit them to be separated.

## The loop

1. **Enumerate** the viable hypotheses (a lag-window version space, refined **lazily**: only the simplest viable
   hypothesis is worth an experiment; a wider history model matters only once the simpler one is falsified).
2. **Predict** what each admissible regime would show — *exactly, and without running it*. This is possible because
   the sources are **roots**: under a regime a clamped source is a constant and a free source is a series already
   held. The observer knows the **question** each experiment asks before it pays for the answer.
3. **Select** the regime that would reveal the most source-history rows the live hypothesis has never seen.
4. **Execute**, verify non-vacuity and temporal provenance, ingest.
5. When nothing is left to explore, **VALIDATE**: predict a regime never executed, run it, and compare exactly.
   A model that has never predicted a regime it has not seen has not been tested, only fitted.
6. A regime whose outcome is already known under the live hypothesis is **inadmissible** — that is not caution,
   it is expense.

## Admissible interventions

**Structure** (ancestry, lags, dissociation): one-step source **pulses**, at every phase of the inferred period.
**Function** (the manifold): **sustained regimes only** — free running plus constant clamps on source subsets.
**Never**: clamping a boundary **tap** as though it were an independent source. A tap is a consequence.

## Verdicts

`IDENTIFIED` · `EQUIVALENCE_CLASS_ONLY` · `INDETERMINATE` · `INSUFFICIENT_HISTORY` ·
`INSUFFICIENT_INTERVENTION_ACCESS` · `OUT_OF_SCOPE` · `CONFOUNDED`

The observer must not force a unique answer when several hypotheses remain observationally equivalent.

## Measured adaptivity and cost

| | development | prospective |
|---|---|---|
| distinct exploration plans | **6** across 15 worlds | plans differ with the source set |
| episodes executed / exhaustive schedule | **150 / 306 (49 %)** | **75 / 90 (83 %)** |

The plan is a **function of the world**: with the gate open the planner clamps the **register**; with the gate shut
it clamps the **write-enable**, because the privileged source set itself differs. That is adaptivity, not a
schedule in costume.

## The defect that retired it

The version space only ever **widens**. When a lag window is contradicted, the observer escalates to a longer
history. It never asks the opposite question — *is this lag necessary?* — and so it can report a **correct but
non-minimal** model, and derive the wrong transducer class from it. See `ACTIVE_OBSERVER_PROSPECTIVE_REPORT.md`.
