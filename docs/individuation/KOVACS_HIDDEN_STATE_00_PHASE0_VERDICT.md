# KOVACS-HIDDEN-STATE-00 — Phase-0 verdict

## Verdict: **GO (conditional)** — advance to Phase-1 design finalisation under human review.

GO means: the concept is theoretically sound, identifiable, feasible, and constraint-compliant, and the
line should proceed to a Phase-1 DEV crossing-rule calibration + power analysis + seal-time lock. GO
does **not** authorise a seal, a prospective family, a new seed namespace, or any execution. Phase 0
stops here for human review, as required.

## Grounds

1. **The logic is sound and the claim is bounded.** `Z`-sufficiency is expressed as an autonomy
   (semiconjugacy) property of a deterministic macro-map; a matched-`Z` divergence under a common
   environment is a constructive falsification of that autonomy. A positive result establishes exactly
   two things — insufficiency of `Z`, and a functionally-active hidden history-dependent DOF — and
   nothing about identity, ownership, reconstruction or heredity (derivation §2, §7).

2. **It is identifiable where the prior line was not.** COUNTERFACTUAL-HISTORY-CORE-00 stopped at
   `NO_MEMORY_FIRST_STAGE` and the mechanism reconciliation returned `UNRESOLVED` precisely because a
   *predictive* body/geometry model did not generalise across worlds in both arms. Kovacs needs no
   predictive model: it is a direct dynamical test of one scalar under the engine's own dynamics, so it
   sidesteps the identifiability wall that stalled the earlier feeding-mediation question.

3. **The matching mechanism is demonstrated; the quantitative match is a scoped Phase-1 item.** On
   already-open DEV worlds a hard-early-feed→starve history drives focal mass up then **down** in 6/6
   worlds (non-monotone overshoot → crossing `m*` from above is robust); the overshoot's declining branch
   descends into the gentle approach's value range in 3/6 worlds (band overlap), but a genuine
   opposite-direction crossing — approach still *ascending* at the shared value — is demonstrated in only
   1/6 (57003, marginal), because the gentle approach itself overshoots-and-declines in low-capacity
   worlds. Tolerance-level matching (both branches within `tol` at `N_min ≥ 12`) was **not** tested (the
   probe used a disclaimed placeholder `m*`). What is proven: matching a single amount provably leaves
   hidden DOF (at fixed **core** mass, ~46 % of area, ~60 % of shape, ~33 % of body mass, ~96–99.6 % of
   memory readouts survive). What remains: a DEV crossing-rule calibration to land the actual match — the
   first Phase-1 task, and the reason the GO is **conditional**.

4. **The design avoids every forbidden move by construction.** The match is produced by
   history/timing, never by body-equalisation surgery; the estimand is a raw signed area of `Z`, never a
   regression residual; there is no decoder/reader battery; `Z` is selected on mechanical criteria and
   is not outcome-adjacent; `m*`, `tol`, `A_min`, `t_ref`, `T_hor` are frozen mechanically and never
   tuned from a response; no new seed namespace or prospective run occurs in Phase 0.

5. **The result is meaningful in either direction.** A clean null is a bounded
   `Z_SUFFICIENT_NOT_ESTABLISHED` (the hidden DOF do not express on `Z` through deep turnover at this
   power) — informative and never a sufficiency proof. A positive is a clean Kovacs demonstration. The
   null-diagnostic schema keeps an invalid match strictly separable from a genuine null.

## Conditions to satisfy before any seal (Phase 1; each a matching-variable or design task only)

- **DEV crossing-rule calibration.** Finalise a frozen per-world schedule family + crossing rule that
  lands both branches within `tol` of the per-world `m*` — opposite-direction where achievable — in a
  pre-declared minimum fraction of the DEV worlds, and demonstrate it across the full DEV set. This uses
  only already-open DEV worlds and the matching variable; it computes no excursion outcome.
- **Power analysis** fixing `N_min` and confirming the horizon, before any confirmatory `N` is chosen.
- **Lock the scalars verbatim** (`tol`, `A_min`, `t_ref`, `T_hor`, deep interval, `M ≤ 0.25`, gate).
- **Independent re-audit** of the frozen package (the project's standard adversarial pass) before a
  FINAL_SEAL is considered, on a fresh disjoint seed namespace.

## Honest risks and how they are handled

- **Velocity-at-match interpretation.** Opposite-direction crossings match `Z` at differing `dZ/dt`; a
  reviewer could call a divergence "guaranteed". This is in fact the classical Kovacs signature (the
  glass protocols also match volume at differing rates), and it is handled by (i) the reference-relative
  hump `K_b` (departure from a *directly-relaxed* system at the same macrostate, not from the other
  branch's velocity), (ii) the one-step discontinuity and identical-history-clone shams, and (iii) an
  equal-age release variant. A Phase-1 refinement may additionally test a near-turning-point (low-`dZ/dt`)
  matching variant to further decouple velocity from hidden state.
- **Turnover may homogenise the hidden DOF** → a genuine null. This is a real and acceptable outcome, not
  a design failure; it is reported as `Z_SUFFICIENT_NOT_ESTABLISHED`, never as sufficiency.
- **Relation to the prior UNRESOLVED line.** Kovacs is not BODY-EQUALIZATION-00: it is probe-free,
  surgery-free, residualisation-free, and tests a dynamical autonomy property rather than a feeding
  mediation. It answers a different, cleaner question.
- **Single substrate / DEV scope.** Generalisation rests only on the independent source worlds; no
  cross-substrate or ownership claim is implied.

## What this verdict does NOT authorise

No seal; no FINAL_SEAL manifest; no prospective seed family; no new namespace; no execution; no
modification of V5, 03G, DIRECTED-CAUSAL-PAIR, or 58xxx artefacts; no ownership/identity/reconstruction
claim. Phase 0 ends here for human review.
